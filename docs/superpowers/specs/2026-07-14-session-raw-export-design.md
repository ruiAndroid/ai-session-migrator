# AI Session Migrator 原始会话导出设计

## 目标

在会话列表最右侧的操作列增加单会话“导出”按钮，让用户可以把当前扫描到的 Codex 会话以原始 `.jsonl` 格式保存到自己选择的位置。

导出必须无损：目标文件与源会话文件逐字节一致，不经过“查看记录”的解析、筛选或截断逻辑。

## 用户体验

- 活跃会话和已归档会话都显示“导出”按钮。
- 按钮位于现有最右侧“操作”列，使用下载图标和“导出”文字，与“查看记录”“归档/激活”等行操作保持一致。
- 点击后打开操作系统原生“另存为”窗口。
- 默认文件名沿用源会话文件名，包括 `.jsonl` 后缀；不使用会话标题生成文件名，避免非法字符、重名和标题变化。
- 用户可以选择任意目标目录并修改文件名，但保存格式保持 `.jsonl`；文件名省略后缀时自动补 `.jsonl`，其他后缀不进入导出 command。
- 用户取消“另存为”时结束操作，不显示错误。
- 导出成功后显示保存路径；导出失败时显示本地化错误，源会话不会被修改。

## 技术方案

采用 Tauri 2 官方文件对话框插件选择目标路径，并由 Rust 后端执行实际复制。

前端负责：

1. 从会话行取得 `codexHome`、`threadId` 和扫描返回的源 `path`。
2. 使用带 `.jsonl` 格式过滤的系统“另存为”窗口获取目标路径，默认名称取源路径的文件名；对话框返回的名称没有后缀时补上 `.jsonl`。
3. 用户确认目标后调用新的导出 command。
4. 展示导出中、成功或失败状态，防止同一行重复触发。

Rust 后端负责：

1. 接收 `codexHome`、`threadId`、源路径和目标路径。
2. 规范化并校验源路径确实位于当前 Codex Home 的 `sessions` 或 `archived_sessions` 目录内。
3. 校验源文件存在、是普通文件且扩展名为 `.jsonl`，同时要求目标路径扩展名为 `.jsonl`。
4. 把源文件逐字节复制到用户选择的目标位置。
5. 返回目标路径和写入字节数，供前端展示完成状态。

文件对话框只决定用户允许写入的位置，不能替代后端对源路径的校验。这样可以防止前端参数被误用为任意本地文件复制入口。

## 数据契约

新增请求：

```text
SessionExportRequest
- codexHome: string
- threadId: string
- sourcePath: string
- destinationPath: string
```

新增结果：

```text
SessionExportResult
- threadId: string
- destinationPath: string
- bytesWritten: number
```

`threadId` 用于让前后端结果与触发导出的会话明确对应；内容复制以已校验的 `sourcePath` 为准，不扫描或猜测其他会话文件。

## 错误处理

- 源路径不属于当前 Codex Home：拒绝导出，提示重新扫描后再试。
- 源文件在扫描后被移动或删除：拒绝导出，提示源会话已不存在。
- 目标目录不可写、磁盘空间不足或复制失败：保留源文件，提示目标路径写入失败。
- 目标文件扩展名不是 `.jsonl`：不执行复制，提示使用原始 JSONL 格式。
- 目标路径与源路径相同：拒绝导出，避免无意义覆盖源会话。
- 系统文件对话框被取消：视为正常结束，不产生错误面板。

导出是只读操作，不要求 Codex 退出，也不创建迁移备份。

## 测试策略

Rust 单元测试覆盖：

- 活跃与归档目录中的 `.jsonl` 可以导出。
- 导出文件与源文件字节完全一致，包括非 UTF-8 安全的任意字节。
- 拒绝 Codex Home 之外的源路径、非 `.jsonl` 文件、缺失文件和源目标相同路径。
- 写入失败返回稳定的错误码，且不修改源文件。

React 测试覆盖：

- 每个会话行都有可访问名称为“导出”的按钮。
- 点击按钮时以原文件名打开“另存为”。
- 取消保存不会调用后端 command，也不会显示错误。
- 确认保存后发送正确请求，并展示成功路径。
- 后端失败时展示本地化错误，并恢复按钮可用状态。

构建验证覆盖 TypeScript/Vite 构建、Vitest 全量测试和 Rust 全量测试。

## 非目标

- 批量导出或 ZIP 打包。
- Markdown、纯文本或其他可读格式转换。
- 导入或恢复导出的会话。
- 修改、清洗或重新序列化会话内容。
- 自动上传、分享或同步导出文件。

## 验收标准

- 用户能从任意活跃或归档会话行打开系统“另存为”窗口。
- 默认文件名与源 `.jsonl` 文件名一致。
- 保存后的文件与源会话逐字节一致。
- 取消操作没有错误提示，失败操作有明确提示。
- 导出过程不修改源会话、Codex 索引或 SQLite 状态。
